# ТЗ лонгрида «Музыка после отъезда»

**Версия:** 1.0 · Июнь 2026  
**Передать:** веб-дизайнеру + разработчику фронтенда  
**Стек (рекомендация):** Next.js (SSG) + D3 / Observable Plot + Scrollama + Framer Motion  
**Деплой:** Vercel или GitHub Pages  
**Данные:** `analysis/output/*.csv` → сконвертировать в JSON скриптом `analysis/export_for_web.py` (написать отдельно)

---

## Обозначения в документе

- **[ТЕКСТ]** — готовая копирайтная вставка, берётся без изменений
- **[ВИЗ]** — спецификация интерактивного блока
- **[ДАННЫЕ]** — исходный CSV/JSON для этого блока
- **[АНИМация]** — поведение при скролле или hover
- **[РЕДАКТОР]** — место, где нужна финальная правка автора
- ОПС-артисты (иностранный агент) упоминаются с пометкой «иноагент»

## Дизайн-токены (кратко)

Полная спецификация — в `STYLE_GUIDE.md`. Здесь — минимум для разработчика.

```css
--black:        #0D0D0D;  /* фон hero */
--paper:        #F0EBE0;  /* фон страницы, карточки */
--ink:          #1A1A1A;  /* основной текст */
--ash:          #3A3A3A;  /* вторичный текст */
--fog:          #8A8A8A;  /* подписи, даты */
--stamp-red:    #C1272D;  /* ОПС, «После», значимые сдвиги */
--stamp-amber:  #D4A017;  /* hover, выделитель, «До» */
--typewriter:   #4A3728;  /* моноширинный текст */
--before:       #4A7FA5;  /* период «До» на графиках */
--after:        #C1272D;  /* период «После» на графиках */

font-heading:   "Playfair Display", serif;
font-body:      "Lora", serif;
font-ui:        "Inter", sans-serif;
font-mono:      "JetBrains Mono", monospace;
```

**Текстура:** SVG-шум opacity 0.03–0.06 поверх всех блоков.  
**Тени карточек:** `box-shadow: 2px 3px 12px rgba(60,40,20,0.18)`

#### Вкладка 1: Словарь тональности

**Заголовок вкладки:**

```
01 / Словарный термометр
```

JetBrains Mono, 15px, `--ink`. Стрелка `›` справа, поворачивается при открытии.

**Содержимое:**

[ТЕКСТ]

```
Компьютер идёт по тексту слово за словом и сверяется со списком: это слово 
обычно звучит позитивно или негативно?

«Любовь» — плюс. «Война» — минус. «Дождь» — ноль.

Итог: индекс от −1 до +1. Если плюсов и минусов поровну — ноль. 
Если минусов больше — отрицательный. У рэпа и рока он почти всегда 
немного ниже нуля — это жанровая норма, не тревожный сигнал.
```

[ВИЗ — мини-интерактив «конструктор строки»]

Блок шириной 100%: 8 слов в ряд, каждое — кликабельный тег. При клике меняет знак (+/−/0) и цвет. Под строкой в реальном времени пересчитывается индекс.

Стартовая строка: `люблю` (+) / `жизнь` (0) / `ненавижу` (−) / `этот` (0) / `серый` (0) / `дождь` (0) / `домой` (0) / `всё` (0)

Текущий расчёт:

```
(pos − neg) / слов со знаком = (1 − 1) / 2 = 0,000
```

Моноширинный, `--typewriter`, обновляется при каждом клике.

**Подпись:** «Попробуйте изменить знак любого слова» — Inter, 13px, `--fog`.

**Ограничения (текст под интерактивом):**

```
Слабое место: «Не рад, что ты ушёл» — для словаря это плюс, потому что «рад» в списке 
позитивных. Отрицание он не видит. Это честное ограничение, о котором мы помним.
```

Lora italic, 15px.

**Содержимое:**

[ТЕКСТ]

```
Алгоритм сам находит «кучи» слов, которые часто появляются вместе, — без

**Важная ремарка:**

```
Мы не искали «патриотизм» или «протест» — машина нашла статистические кластеры.

## Scrollytelling

Рекомендуемая библиотека: **Scrollama** (JS) или `react-scrollama`. Логика:

- Каждый «шаг» = `<Step>` с data-index
- При входе step в viewport: обновить стейт → нужный график проявляется (blur 6px → 0, opacity 0.3 → 1, 500ms)
- Sticky-контейнер графика: `position: sticky; top: 10vh; height: 80vh`

## Адаптив

- `>1280px`: две колонки в scrollytelling
- `768–1280px`: одна колонка, график под текстом шага
- `<768px`: мобильная версия, картотека = вертикальный стек 1 в ряд

## Производительность

- D3-графики монтировать только при первом входе в viewport (IntersectionObserver)
- JSON-данные: `import` статически (Next.js SSG) или `fetch` на клиенте
- Шрифты: `font-display: swap`, только нужные веса
