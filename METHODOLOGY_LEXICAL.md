# Методология: лексический анализ тональности

Метод 1 из исследования «Музыка после отъезда».  
Скрипт: `analysis/lexical_sentiment.py`  
Результаты: `analysis/output/`

---

## 1. Цель

Численно сравнить тональность текстов российских музыкантов-эмигрантов в двух периодах — до и после релокации, — используя словарный (лексический) подход без нейросетевых компонентов.

---

## 2. Источник данных

| Файл | Содержание |
|---|---|
| `data/songs.jsonl` | Тексты треков, собранные через Genius API |
| `data/sample_musicians.csv` | Профили 50 артистов с датами релокации и флагом `genius_ok` |
| `data/rusentilex_2017.txt` | Словарь тональности RuSentiLex 2017 |

В анализ включаются только артисты с `genius_ok = TRUE` (24 из 50). Остальные исключены из-за недостаточного числа треков (< 10 до или < 10 после) или отсутствия на Genius.

---

## 3. Словарь тональности: RuSentiLex 2017

**RuSentiLex** (Loukachevitch & Levchik, 2016) — стандартный словарь для русскоязычного NLP. Содержит ~16 000 записей с разметкой:
- `positive` → +1
- `negative` → −1
- `neutral` → 0
- `positive/negative` → пропускается (контекстно-зависимые слова)

В словарь добавляются и словарные формы, и леммы каждой записи, что увеличивает покрытие без необходимости полного морфологического анализа. Итого в рабочем словаре ≈ 10–13 тысяч ключей (однословных).

**Ограничение.** Словарь не учитывает контекст: отрицания, иронию и переносные значения. Это принципиальное ограничение метода 1, устранимое методами 2 и 3.

---

## 4. Предобработка текстов

1. **Удаление служебных элементов Genius:** строки вида `N Contributors … Lyrics` и заголовки секций `[Куплет 1]`, `[Припев]`, `[Verse]` и т. п.
2. **Токенизация:** извлекаются только кириллические последовательности символов (регулярное выражение `[а-яёА-ЯЁ]+`), что отсеивает транслит, числа, знаки препинания и иноязычные вставки.
3. **Нормализация:** все токены приводятся к нижнему регистру и сопоставляются со словарём.

Треки с менее чем 20 кириллическими словами исключаются из анализа.

---

## 5. Метрики тональности

Для каждого трека вычисляются следующие показатели:

| Метрика | Формула | Интерпретация |
|---|---|---|
| `sentiment_ratio` | (pos − neg) / total_words | **Основная.** Относительный перевес позитивных слов во всём тексте |
| `polarity` | (pos − neg) / (pos + neg) | Полярность среди тональных слов |
| `pos_ratio` | pos / total_words | Доля позитивных слов |
| `neg_ratio` | neg / total_words | Доля негативных слов |
| `tonal_density` | (pos + neg) / total_words | Насыщенность тональными словами |

`sentiment_ratio` ∈ [−1, 1]; значение 0 означает нейтральный или сбалансированный текст.

---

## 6. Разметка периодов

Границей периодов служит поле `reloc_date` из `sample_musicians.csv`.  
- **До** — треки, выпущенные *строго до* даты релокации (`period = "before"`).  
- **После** — треки, выпущенные *начиная с* даты релокации (`period = "after"`).

Логика разметки подробно описана в `README.md` (раздел «Разметка периодов»).

---

## 7. Два метода сравнения

### Метод A — сбалансированная выборка (Balanced)

Для каждого артиста берётся одинаковое количество треков из обоих периодов:

```
N = |after_tracks|
before_sample = N самых новых треков из периода "до"
```

**Обоснование.** Исключает дисбаланс выборок, при котором один период доминирует численно и может искажать агрегированные результаты. При нескольких артистах с большим «до» и малым «после» среднее по «до» будет более надёжным.

**Ограничение.** Теряем часть информации из «до». Самые ранние треки (возможно, наиболее показательные для «базового» периода) могут не попасть в выборку.

### Метод B — все имеющиеся треки (All)

Сравниваются все треки «до» и «после» без ограничений.

**Обоснование.** Максимальная статистическая мощность; позволяет увидеть разницу при достаточном объёме данных.

**Ограничение.** Артисты с большим числом треков «до» вносят непропорциональный вклад в общую картину.

---

## 8. Статистические тесты

| Тест | Зачем |
|---|---|
| **Mann–Whitney U** | Непараметрический; не предполагает нормальности распределений тональности |
| **Cohen's d** | Размер эффекта: d < 0.2 — пренебрежимо малый, 0.2–0.5 — малый, 0.5–0.8 — средний, > 0.8 — большой |

Тесты проводятся дважды:
1. На всей совокупности (aggregated).
2. Отдельно для каждого артиста.

Уровень значимости: α = 0.05. Значимые результаты помечаются звёздочками (* p < 0.05, ** p < 0.01, *** p < 0.001).

## 10. Предварительные результаты

*(На момент запуска — частичная выборка; датасет продолжает наполняться)*

| Показатель | Метод A (Balanced) | Метод B (All) |
|---|---|---|
| Среднее «до» | −0.0074 | −0.0045 |
| Среднее «после» | −0.0055 | −0.0055 |
| Cohen's d | 0.055 (пренебрежимо) | −0.034 (пренебрежимо) |
| p-value | 0.644 (n.s.) | 0.036 (*) |

**Интерпретация.** Оба периода показывают слабо-негативную тональность (доминируют негативные слова). Метод B фиксирует статистически значимую, но практически пренебрежимо малую разницу (d ≈ −0.03). Метод A разницы не обнаруживает. Выводы следует уточнить после завершения сбора текстов.

---

## 11. Ограничения метода

1. **Контекст не учитывается.** Отрицание («не рад»), ирония, метафора не обрабатываются.
2. **Мат и разговорная лексика.** Значительная часть тропов рэп-текстов отсутствует в словаре.
3. **Мультиязычные вставки.** Треки с украинским или английским текстом теряют часть токенов при фильтрации кириллицы.
4. **Небольшой период «после».** У ряда артистов период «после» охватывает 2–4 года; у некоторых — 10+ лет «до». Сбалансированная выборка частично компенсирует это.

```bash
python3 analysis/lexical_sentiment.py
```
