# Музыка после отъезда: как изменилось творчество российских артистов в эмиграции
Исследование трансформации текстов российских музыкантов после эмиграции в 2022-2023 годах. Используются автоматизированные методы анализа. 

## Определения
### Музыкант
Автор музыкального текста. В этой работе мы не учитываем авторов и исполнителей инструментальной или электронной музыки, так как субъектом исследования являются только тексты.
### Миграция
Физическая релокация в другую страну на продолжительный срок. Мы понимаем, что музыкант релоцировался только если в публичном поле есть источник: интервью, пост музыкана, прочие достоверные сведения. 
### Особый правовой статус (ОПС)
Иностранный агент, экстремист, террорист. Правила по определению статуса и работе с ними описаны в LEGAL_COMPLIANCE.

## Контекст
Часть артистов покинула страну во время так называемой "пятой волны российской эмиграции", начавшейся в 2022 году. 

## Исследовательский вопрос
Как изменилась тональность текстов российских артистов после того, как они эмигрировали. 

## Методология
### Методы:
1. Лексический анализ тональности (основной)
    Каждому слову текста сопоставляется тональный балл из заранее составленного словаря (например, RuSentiLex). Тональность трека выводится как агрегат по всем словам, что позволяет сравнивать периоды "до" и "после" численно.
2. Тематическое моделирование (дополнительный)
    Статистический метод, выявляющий латентные темы в корпусе текстов без предварительной разметки (LDA, BERTopic). Позволяет установить, какие темы характерны для каждого периода и как тематический состав текстов артиста меняется после релокации.
3. Трансформерные модели (дополнительный)
    Нейросетевые языковые модели (ruBERT, ruRoBERTa), предобученные на больших корпусах русского языка. В отличие от лексического подхода, учитывают контекст слова в предложении, что повышает точность анализа тональности при наличии иронии, отрицаний и переносных значений.

### Правила
Минимальная единица анализа – это текст. Выше – артист и его период (до или после релокации).
Мы документируем источники данных о релокации и присвоении ОПС. Для каждого артиста индивидуально. На каждого артиста нужна ссылка, подтверждающая его релокацию. 
### Сбор данных
1. Определение выборки
    Собираем перечень артистов (цель – 50+), подходящих под критерии. В этом датасете должна быть информация:
    1. Псевдоним
    2. Реальное имя
    3. Дата релокации
    4. Страна релокации
    5. Правовой статус
    6. Источник о релокации
    7. Источник о правовом статусе
    8. Источник о стране релокации
    К датасету ставится требование об однозначности. Не допускаются пересечения между артистами и коллективами. Нас интересуют авторы текста.
    Страна релокации – первое постоянное место жительства (или штат, если доступно) в периоде релокации. Без учета транзитных стран и последующих переездов.
2. Сбор текстов
    Парсинг Genius API. Собираем максимум что можем по всем артистам. Фильтровать будем уже на этапе анализа. 

## Источники текстов
Источник – Genius.
У него есть официальный API, и он является стандартом в NLP-исследованиях текстов.

## Критерии отбора музыкантов
1. Опубликовано как минимум 10 текстов до и 10 текстов после миграции.
2. Есть источники, подтверждающие, что человек релоцировался в период 2022-2023 годов (интервью, пост музыкана, прочие достоверные сведения).
3. Музыкант до релокации постоянно проживал и вел деятельность в России – его можно отнести именно к российской культуре.
4. Релокация не связана с краткосрочными событиями. Это именно переезд на длительное время – от года.

## Разметка периодов
Часто мы не располагаем открытыми данными о точных датах пересечения границы, это нормально. Если нет четкой даты, мы берем минимальную дату из известных. Например, если в источнике "релоцировался в марте 2022" – дату релокации ставим 1 марта 2022. Тексты, что были выпущены до этой даты будут отмечены как "До", а тексты, начиная с этой даты "После". 
Мы упрощаем модель и не учитываем повторные визиты в Россию.
В анализе для сопоставимости периодов для каждого музыканта берётся равное количество треков «до» и «после»: N = min(число треков до, число треков после). Из каждого периода выбираются N самых новых по дате выпуска. Так исключается длинный хвост ранних лет карьеры, который искажал сравнение при подходе «все треки до vs все после».

## Веб-лонгрид

Исследование опубликовано в виде интерактивного лонгрида на Next.js. Исходники — в папке `web/`.

### Быстрый старт

```bash
# 1. Сгенерировать JSON-данные из CSV-результатов анализа
python3 analysis/export_for_web.py

# 2. Установить зависимости и запустить dev-сервер
cd web && npm install && npm run dev
# Откроется http://localhost:3000
```

### Production-сборка

```bash
cd web && npm run build
# Статика создаётся в web/out/
```

**Полная инструкция по деплою на VPS (git clone → домен → HTTPS):** [DEPLOY.md](DEPLOY.md)

Кратко: собрать `web/out/`, отдать через nginx, SSL через certbot.

```bash
# Проверить статику локально
npx serve out -p 3000
```

## Ограничения работы
Большинство из исследуемых артистов получили "особые правовые статусы". Теперь, согласно закону, они экстремисты, террористы, иностранные агенты. Любое публичное упоминание (в т.ч. академические работы) и освещение их деятельности ограничены законом. 
Требуется с особым вниманием относиться к цитатам и упоминаниям, соблюдать все требования законодетельства. Придерживаться нейтрального, академического языка. При любом их упоминании ссылаться на особые статусы. Опасаться позитивной оценки их деятельности и не пропагандировать их идеи.
