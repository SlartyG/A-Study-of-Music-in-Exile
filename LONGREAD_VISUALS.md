# Визуализации лонгрида: что показывать и чем заменить PNG

Документ связывает **методологию проекта**, **STYLE_GUIDE.md** и **актуальные выводы** (`FINDINGS.md`). Цель — не переносить 40 PNG в статью, а собрать **8–10 интерактивных блоков**, которые ведут читателя по сюжету.

Техническая реализация — Observable Plot / D3 по рекомендации из STYLE_GUIDE. Данные — из `analysis/output/*.csv` (сбалансированная выборка).

---

## Принципы отбора

1. **Один график — один тезис.** Если блок не поддерживает конкретное предложение в тексте, его нет на странице.
2. **Честность выводов.** На сбалансированной выборке нет значимого общего сдвига тональности; зато есть сдвиг тем. Лонгрид не должен визуально кричать «стало мрачнее», если цифры этого не показывают.
3. **Эстетика «Досье».** Фон `#F0EBE0`, до — `#4A7FA5`, после — `#C1272D`, без rainbow и 3D. См. STYLE_GUIDE, раздел «Инфографика».
4. **Интерактив только там, где он отвечает на вопрос.** Hover с именем артиста — да. «Покрутите 3D-глобус» — нет.
5. **Методология — в accordion**, не в основном потоке. Полный зоопарк графиков для ревьюера — в репозитории, не на сайте.

Последовательность секции «Результаты» — **пересмотрена** относительно черновика в STYLE_GUIDE с учётом балансировки:

| Шаг | Тезис для читателя | Интерактив | Исходный PNG |
|---|---|---|---|
| 1 | Жанр и корпус: тексты и до, и после слегка «в минусе» — это норма для рэп/рока, не эффект отъезда | Violin «до / после» (лексический метод) | `1_violin_overall.png`, `final_2` (только панель «все») |
| 2 | В среднем по 21 артисту **сдвига тональности нет** (p > 0,05, d ≈ 0) | Компактная таблица-«выписка» + анимированные числа | `final_1_summary_table.png` |
| 3 | Зато траектории **разные**: у кого-то плюс, у кого-то минус | Dot-plot Δ по артистам, hover, подсветка p < 0,05 | `5_delta.png`, упростить `final_3` |
| 4 | **Темы меняются** — это устойчивее, чем «настроение» | Парный bar «темы выросли / упали» + тултип с топ-словами | `tm_2`, `tm_3`, `tm_8` |

**Убрать из главной истории** (не подтверждается данными или дублирует):

- «С ОПС vs без ОПС ведут себя по-разному» как отдельный шаг — на балансированной выборке **групповых различий нет** (p > 0,7). Фильтр ОПС оставить в картотеке, не как центральный график.
- Два метода тональности рядом в основном потоке — перегруз. ruBERT — в accordion «Как считали».

## Секции лонгрида × визуализации

Согласовано со структурой STYLE_GUIDE; уточнения в скобках.

### 1. Hero (тёмный фон)

- Анимированные счётчики: `21` артист · `1 598` треков · `2022–2023`
- Без графиков
- Шрифты и зерно — по STYLE_GUIDE

### 2. Что произошло

- Только текст + одна статичная схема «отбор артистов» (SVG): Genius → фильтры → 21 артист
- Без PNG

### 3. Как мы считали (accordion, закрыт по умолчанию)

Три вкладки по `POPULAR_METHODS.md`:

| Вкладка | Контент | Визуал |
|---|---|---|
| Словарь | Пример расчёта на 8 словах | Мини-таблица + ползунок «добавить слово» |
| LDA | Аналогия «кучи писем» | Статичная иллюстрация или 3 кликабельные «кучи» |
| ruBERT | «Не рад, что ты ушёл» | Две колонки: словарь vs модель |
| Балансировка | N самых новых треков | **Интерактив-схема**: полоска «15 лет до» сжимается до N треков |

### 4. Артисты (картотека)

Центральный UX-элемент. Заменяет десятки bar-chart PNG.

- Фильтры: Все / С ОПС / Без ОПС / страна
- Карточка: псевдоним, дата, страна, штамп ОПС, n до/после, Δ лекс + значимость
- Клик → боковая панель: sparkline тональности, топ-темы, 2–3 цифры
- Данные: `final_per_artist_lex.csv`, `final_per_artist_bert.csv`, `topic_scores.csv`

### 5. Результаты (scrollytelling)

См. таблицу «Главная история» выше. Desktop: текст слева, график справа (sticky). Mobile: график под абзацем.

### 6. Темы

Объединить в одну секцию с шагом 4 scrollytelling или вынести отдельным разделом после тональности — на усмотрение монтажа. Главное: **темы — единственный устойчивый групповой сдвиг**.

### 7. Выводы (тёмный фон)

- Шесть тезисов текстом, без графиков
- Цифры моноширинным (`JetBrains Mono`)

### 8. Оговорки

- Мелкий серый текст: балансировка, ограничения RuSentiLex/ruBERT, LEGAL_COMPLIANCE

## Спецификации интерактивов (кратко)

### Violin «Тональность»

- Данные: все треки из `sentiment_scores.csv`
- X: До / После; Y: sentiment_ratio
- Цвета: `#4A7FA5` / `#C1272D`, opacity 0.45
- Hover: значение трека (опционально обезличенно)
- Подпись: «Каждая полоска — распределение ~800 треков»

### Таблица-сводка

- 2 строки для читателя: лексический + ruBERT (группа «все артисты»)
- Опционально свернуть блок «С ОПС / Без ОПС» — не акцентировать

### Delta dot-plot

- Сортировка по Δ
- Цвет: Δ < 0 → красный, Δ > 0 → синий (или нейтральный серый если |Δ| мала)
- Звёздочка / размер точки при p < 0,05
- Подпись: «Зелёный/красный — направление; звезда — p < 0,05»

### Scatter до/после

- Оси: mean_before × mean_after по артисту
- Диагональ: «без изменений»
- Форма: квадрат = ОПС, круг = нет

### Таймлайн

- Две серии (календарные до/с 2022)
- Band ± SEM
- В 2022: особая подпись «итог / старт периода»
- Сноска: «Не дата отъезда каждого артиста, а год выпуска трека»

### Темы (diverging bar)

- 10 тем → 5 с ростом, 5 с падением (или топ-3 / топ-3)
- Человеческие ярлыки вручную (не «сука / хей / нахуй» в заголовке — в тултипе)
- χ² и p — одна строка моноширинным под графиком

## Что поправить в STYLE_GUIDE.md

При реализации лонгрида стоит обновить STYLE_GUIDE (раздел «Структура лонгрида», §5):

1. Убрать или смягчить шаг «ОПС vs без ОПС» в scrollytelling — данные не поддерживают.
2. Добавить шаг «В среднем сдвига нет» перед delta по артистам.
3. Поднять секцию «Темы» на уровень главного вывода, не хвоста.
4. Зафиксировать палитру timeline: в STYLE_GUIDE уже `#4A7FA5` / `#C1272D` — совпадает с логикой до/после.
5. Явно написать: **PNG из `analysis/output/` не вставляются** — только источник данных для вёрстки.
