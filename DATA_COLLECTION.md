# Сбор данных: методология и техническая реализация

Документ описывает, как формировалась выборка артистов и как собирались тексты с Genius.

---

## Часть 1. Формирование выборки артистов

### Источники

Выборка собиралась вручную по открытым источникам. Основные из них:

| Источник | Что использовали |
|---|---|
| [msk1.ru — список уехавших деятелей культуры](https://msk1.ru/text/culture/2022/12/20/71911313/) | Первичный список артистов с датами и странами |
| [theblueprint.ru — карта эмигрировавших музыкантов](https://theblueprint.ru/culture/music/karta-dop) | Расширенный список + страны релокации |
| [i-m-i.ru — музыканты покинувшие Россию](https://i-m-i.ru/post/musicians-on-leaving-russia) | Независимый список с источниками |
| [i-m-i.ru — релокация](https://i-m-i.ru/post/relocation) | Дополнительные данные |
| Meduza, 76.ru, svoboda.org, novayagazeta.eu | Верификация отдельных артистов |
| Официальные соцсети и сайты артистов | Верификация города, даты |
| Минюст РФ (реестр иноагентов) | Правовой статус |

### Критерии включения

Артист включался в выборку при выполнении всех условий:

1. **Минимум 10 треков на Genius до и 10 после** даты релокации (по `genius_songs_before` ≥ 10 и `genius_songs_after` ≥ 10).
2. **Подтверждённая релокация** в 2022–2023 гг. — есть хотя бы один публичный источник (интервью, пост, статья).
3. **До релокации постоянно жил и работал в России** — артист относится к российской культуре.
4. **Переезд на длительный срок** (≥ год), не краткосрочные гастроли или временный отъезд.

### Дата релокации

Если известен только месяц — берётся первый день: «март 2022» → `2022-03-01`. Это минимальная оценка, смещающая в сторону более широкого «после». Повторные визиты в Россию не учитываются.

### Страна релокации

Фиксируется **первое постоянное место жительства** после отъезда. Транзитные страны и последующие переезды не учитываются. Например, Вася Обломов: первые месяцы — Египет, потом США → фиксируем США.

### Артисты из коллективов

Если несколько авторов текстов из одного коллектива (Би-2, Little Big, Каста) уехали, каждый фиксируется отдельной строкой. При этом `genius_id` у них **общий** (страница группы), и это отмечено в поле `примечания`. На этапе анализа эти строки будут рассматриваться как один источник треков.

### Итоговый датасет

Файл: `data/sample_musicians.csv`

**50 записей**, из которых **32 прошли критерии** (`genius_ok = TRUE`). Исключённые остаются в файле с флагом `FALSE` и причиной в `примечания`.

Поля датасета:

| Поле | Описание |
|---|---|
| `id` | Порядковый номер |
| `псевдоним` | Псевдоним артиста |
| `реальное_имя` | Настоящее имя |
| `основная_группа` | Группа или «сольный» |
| `дата_релокации` | Дата в формате `YYYY-MM-DD` |
| `страна_релокации` | Первая страна после отъезда |
| `город_релокации` | Город (если установлен) |
| `правовой_статус` | иноагент / нет |
| `дата_правового_статуса` | Дата присвоения статуса |

## Часть 2. Сбор метаданных треков (Phase 1)

### Инструмент

Официальный [Genius API](https://docs.genius.com). Токен хранится в `.env` (`GENIUS_ACCESS_TOKEN`).

### Что собирали

Для каждого артиста с `genius_ok = TRUE` делались постраничные запросы к эндпоинту `/artists/{id}/songs` (50 треков на страницу, сортировка по дате выпуска).

Для каждого трека сохранялись:

```jsonl
{
  "song_id": 13520340,
  "title": "Спящая красавица (Sleeping Beauty)",
  "artist_genius_id": 638315,
  "artist_pseudonyms": ["Монеточка"],
  "csv_row_ids": ["3"],
  "reloc_date": "2022-02-01",
  "release_date_components": {"year": 2023, "month": 4, "day": 7},
  "release_date_for_display": "April 7, 2023",
  "period": "after",
  "url": "https://genius.com/Monetochka-sleeping-beauty-lyrics",
  "genius_path": "/Monetochka-sleeping-beauty-lyrics"
}
```

Поле `period`:
- `"before"` — дата выпуска раньше `reloc_date`
- `"after"` — дата выпуска начиная с `reloc_date`
- `"unknown"` — дата отсутствует

### Дедупликация

Некоторые треки коллабораций попадают одновременно в список нескольких артистов (например, совместный трек Noize MC и Oxxxymiron). Такие записи попадают в `song_list.jsonl` дважды. На этапе Phase 2 и при финальной обработке дедупликация производится по `song_id`.

### Результат

Файл: `data/song_list.jsonl`  
Строк: ~6018, уникальных `song_id`: **4440**

## Часть 3. Сбор текстов (Phase 2)

### Проблема: Cloudflare

Genius защищён Cloudflare. Прямые HTTP-запросы (в т.ч. с токеном API) возвращают 403. Решение — использовать реальный браузер с уже пройденной капчей.

### Архитектура

```
browse CLI daemon
  └── Chrome (headed, с Cloudflare-куками)
        └── Chrome DevTools Protocol (CDP)
              └── Playwright (asyncio)
                    ├── Tab 1 ─── Genius page N
                    ├── Tab 2 ─── Genius page M
                    ├── Tab 3 ─── Genius page K
                    └── Tab 4 ─── Genius page J
```

1. **`browse` CLI** (Cursor-плагин) запускает Chrome как headed-браузер с постоянным профилем. Cloudflare проходится один раз вручную (при первом запуске).
2. **Playwright** подключается к уже запущенному Chrome через CDP: `playwright.chromium.connect_over_cdp(ws_url)`. Новый Chrome не создаётся — новой капчи нет.
3. **N параллельных вкладок** (по умолчанию 4) каждая независимо обходит свою часть очереди.
4. **Семафор** `asyncio.Semaphore(min(workers, 4))` ограничивает одновременные навигации — Chrome деградирует при ≥8 одновременных тяжёлых страниц.

### Нахождение CDP-порта

Скрипт автоматически находит порт так:
1. Читает PID демона из `/tmp/browse-{session}.pid`
2. Через `ps -A` находит дочерний процесс Chrome с аргументом `--remote-debugging-port=N`
3. Делает HTTP-запрос на `localhost:N/json/version` и получает WebSocket URL
4. Если сессия `default` не найдена — перебирает все `browse-*.pid` в `/tmp/`

### Извлечение текста со страницы

Genius рендерит тексты серверно (SSR). После `DOMContentLoaded` элементы `[data-lyrics-container="true"]` уже присутствуют в DOM.

JavaScript, исполняемый в странице:

```javascript
() => {
    const containers = document.querySelectorAll('[data-lyrics-container="true"]');
    const getText = el => {
        let t = '';
        for (const n of el.childNodes) {
            if (n.nodeName === 'BR') { t += '\n'; }
            else if (n.nodeType === 3) { t += n.textContent; }
            else if (!['SCRIPT','STYLE','A'].includes(n.nodeName)) { t += getText(n); }
        }
        return t;
    };
    const lyrics = Array.from(containers).map(getText).join('\n').trim();
    return { containers: containers.length, lyrics };
}
```

Логика: рекурсивно обходим DOM-дерево каждого контейнера, `<BR>` превращаем в `\n`, ссылки (`<A>`) — пропускаем (это аннотации Genius, не часть текста).

### Обработка 404

Genius возвращает страницу с заголовком «Burrr! | Genius» для несуществующих URL. Это **не** bot-detection — в обычном браузере та же страница. Такие треки сразу помечаются `has_lyrics: false` без ожидания.

### Resumability

- Каждый обработанный трек сразу записывается в `data/songs.jsonl` и буфер сбрасывается (`flush`).
- При перезапуске все `song_id` из уже записанных строк загружаются в множество `done` — повторно не обрабатываются.
- Скрипт переносит прерывания безопасно: даже если упал на середине, следующий запуск продолжает с того места.

### Оптимальное число воркеров

Экспериментально: **4 воркера** дают ~1 трек/с (ETA ~1.5ч для 5800 треков).

| Воркеров | sem | Скорость | Почему |
|---|---|---|---|
| 3 | 2 | 0.8/с | Исходный запуск |
| 4 | 4 | ~1.0/с | Оптимум |
| 15 | 15 | 0.5/с | Chrome перегружен, рендер деградирует |

Увеличение воркеров сверх ~4–5 приводит к замедлению: Chrome делит CPU между рендерами тяжёлых страниц Genius и теряет в скорости на каждую.

### Формат выходных данных

Файл: `data/songs.jsonl`

```jsonl
{
  "song_id": 13520340,
  "title": "Спящая красавица (Sleeping Beauty)",
  "artist_genius_id": 638315,
  "reloc_date": "2022-02-01",
  "period": "after",
  "url": "https://genius.com/Monetochka-sleeping-beauty-lyrics",
  "has_lyrics": true,
  "lyrics": "Строфа первая...\nСтрофа вторая...",
  "worker": 0
}
```

### Запуск

```bash
# Предварительно — запустить Chrome через browse CLI и открыть genius.com
# (один раз пройти Cloudflare вручную если требуется)
/path/to/browse open "https://genius.com"

# Phase 1 + Phase 2
python3 -u parse_lyrics.py

# Только Phase 2 с кастомным числом воркеров
python3 -u parse_lyrics.py --lyrics --workers 4
```
